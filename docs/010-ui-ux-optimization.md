## 一、标注 Tab — 信息层级重组

### 问题
- 文件名、词性过滤 chips、统计条、排序按钮四个区域垂直堆叠，占据大量空间
- 统计条使用 4px 高进度条，信息密度低
- 排序按钮占一行但内容只有三个短按钮

### 建议

**方案 A（推荐）：合并 chips + 统计条**

```
┌─ README.md ────────────────────────────────────┐
│  [n 142]  [v 89]  [a 45]  [• 23]            │
│  ── 频率↓ ── A→Z ── 词性 ──                   │
├───────────────────────────────────────────────┤
│  词汇列表...                                   │
└───────────────────────────────────────────────┘
```

- 将统计数字以徽章形式直接显示在 chips 上（n 142 / v 89 / a 45 / • 23）
- 点击 chip 切换过滤（同时显示/隐藏对应词性的统计）
- 排序按钮改为紧凑行（去掉边框圆角按钮，改为带下划线的文本链接样式）

**方案 B：固定统计条 + 浮动排序**

- 统计条保持原始进度条样式但高度增加到 6-8px
- 排序按钮嵌入到词汇列表的头部（`position: sticky; top: 0`）
- 文件名和 chips 合并到同一行

### 影响范围
- `panel.html` L28-61：重组 annotate-header、stats-bar、freq-header
- `panel.css` L61-106：调整 stats-bar、chip 样式
- `panel.js` L99-184：`buildWordFrequency`、`renderStats`

## 二、词典 Tab — 详情视图工具栏优化

### 问题
- 详情视图的 `detail-toolbar` 在一行内塞入：搜索框 + 25/50/100 按钮 + 总数
- 当搜索框获得焦点时横向空间不足
- 词条列表缺少视觉交替（无斑马纹）

### 建议

```
┌─ 我的词典 ─────────────────────────────────────┐
│  [← 返回]                        [EN] [+ 新增] │
├───────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────┐  │
│  │  搜索词汇...                            │  │
│  └─────────────────────────────────────────┘  │
│  ── [25] [50] [100] ───────────── 共 1,024 词 │
├───────────────────────────────────────────────┤
│  word1  n adj                                  │
│  word2  v                                     │
│  ... (斑马纹交替背景色)                        │
├───────────────────────────────────────────────┤
│  [◀] [1] [2] [3] ... [42] [▶]                │
└───────────────────────────────────────────────┘
```

- 搜索框独占一行，具有完整宽度
- 分页大小和总数放在搜索框下方的辅助行
- 词条列表添加斑马纹（奇数行浅色背景）
- 返回按钮和语言标签合并到 topbar

### 影响范围
- `panel.css` L189-261：detail-toolbar、entries-list
- `panel.html` L131-148：detail 视图结构调整
- `panel.js` L438-574：`openDictDetail`、`renderEntries`

## 三、批量 Tab — 信息密度优化

### 问题
- 子标签 [文件视图] / [处理结果] 本质上是两个不同的视图状态，切换成本较低
- 处理结果视图包含三个可折叠区域（文件列表 + 统计 + 聚合频率），层级复杂
- 聚合频率折叠头部的 JavaScript 控制逻辑与渲染绑定，可用 CSS 纯实现

### 建议

**3.1 操作栏整合**

当前清除按钮放在操作栏右侧，处理按钮分布在文件树的每个节点上。建议：
- 将"处理"按钮统一放到操作栏（处理选中 / 处理全部）
- 文件树仅用于浏览和选择，不触发操作
- 减少每个节点上的按钮干扰

**3.2 聚合频率默认折叠**

- 聚合频率区域默认收起
- 仅在有结果时才展开提示（闪烁 / 高亮）
- 使用纯 CSS `checkbox` hack 或 `<details>` 元素替代 JavaScript 状态管理

**3.3 折叠箭头复用 chevron 图标**

- 当前使用 ▶ ▼ 等 Unicode 符号，风格统一后与 SVG 图标系统的 `chevronRight`/`chevronDown` 保持一致

### 影响范围
- `panel.css` L582-757：batch tab 全部样式
- `panel.js` L1034-1297：batch 渲染函数
- `panel.html` L152-220：batch tab 结构

## 四、全局改进

### 4.1 增加 padding 空间

当前 `padding: 8px`，在较宽屏幕上偏紧凑。建议：
- 基础 padding 增加到 10-12px
- Tab 内容区域之间使用分割线（现有 border-bottom 样式已足够）

### 4.2 Overlay 一致性

当前 4 个 overlay 基本一致，但：
- 编辑 overlay 的 `max-width: 320px` 在宽屏偏窄，可增至 360px
- Toast 位置（bottom: 8px）与 word-tooltip（bottom: 48px）不重叠但间距小
- Overlay 面板的 box-shadow 可以统一

### 4.3 微交互动画

- 列表条目淡入：添加 `animation: fadeIn 150ms` 过渡
- 翻页时内容闪烁：添加 `opacity` 过渡
- 过滤切换时统计数字更新动画：CSS `transition` 即可

```css
@keyframes fadeIn {
  from { opacity: 0; transform: translateY(4px); }
  to   { opacity: 1; transform: translateY(0); }
}
.freq-row, .entry-row, .batch-agg-row {
  animation: fadeIn 150ms ease-out;
}
```

### 4.4 键盘导航和焦点管理

- 确保 Tab 键顺序匹配视觉顺序
- Overlay 打开时焦点自动移到第一个可交互元素（当前已有部分实现）
- Overlay 关闭时焦点回到触发按钮
